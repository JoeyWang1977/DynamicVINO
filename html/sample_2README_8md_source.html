1 # Interactive Face Detection Sample {#InferenceEngineInteractiveFaceDetectionSampleApplication}
2 
3 This sample showcases Object Detection task applied for face recognition using sequence of neural networks.
4 Async API usage can improve overall frame-rate of the application, because rather than wait for inference to complete,
5 the app can continue doing things on the host, while accelerator is busy.
6 Specifically, this demo keeps three parallel infer requests for Age Gender Head Pose and Emotions detection that run simultaneously.
7 
8 Other demo objectives are:
9 * Video as input support via OpenCV
10 * Visualization of the resulting face bounding boxes from Face Detection network
11 * Visualization of age gender, head pose and emotion recognition information for each detected face
12 * OpenCV is used to draw resulting bounding boxes, labels, etc, so you can copy paste this code without
13 need to pull Inference Engine samples helpers to your app
14 
15 ### How it works
16 
17 On the start-up the application reads command line parameters and loads one, two, three or four networks depending on -d... options family to the Inference
18 Engine. Upon getting a frame from the OpenCV's VideoCapture it performs inference of frame detection network, then performs three simultaneous inferences
19 using Age Gender, Head Pose and Emotions detection networks (if those specified in command line) and displays the results.
20 
21 New "Async API" operates with new notion of the "Infer Request" that encapsulates the inputs/outputs and separates *scheduling and waiting for result*,
22 next section. And here what makes the performance look different:
23 1. In the default ("Sync") mode the frame is captured and then immediately processed, below in pseudo-code:
24 ```cpp
25  while(true) {
26  capture frame
27  populate FaceDetection InferRequest
28  wait for the FaceDetection InferRequest
29  populate AgeGender InferRequest using dyn batch technique
30  populate HeadPose InferRequest using dyn batch technique
31  populate EmotionDetection InferRequest using dyn batch technique
32  wait AgeGender
33  wait HeadPose
34  wait EmotionDetection
35  display detection results
36  }
37 ```
38  So, this is rather reference implementation, where the new Async API is used in the serialized/synch fashion.

41 ## Running
42 
43 Running the application with the <code>-h</code> option yields the following usage message:
44 ```sh
45 ./interactive_face_detection -h
46 InferenceEngine:
47  API version ............ <version>
48  Build .................. <number>
49 
50 interactive_face_detection [OPTION]
51 Options:
52 
53  -h Print a usage message.
54  -i "<path>" Optional. Path to an video file. Default value is "cam" to work with camera.
55  -m "<path>" Required. Path to an .xml file with a trained face detection model.
56  -m_ag "<path>" Optional. Path to an .xml file with a trained age gender model.
57  -m_hp "<path>" Optional. Path to an .xml file with a trained head pose model.
58  -m_em "<path>" Optional. Path to an .xml file with a trained emotions model.
59  -l "<absolute_path>" Required for MKLDNN (CPU)-targeted custom layers.Absolute path to a shared library with the kernels impl.
60  Or
61  -c "<absolute_path>" Required for clDNN (GPU)-targeted custom kernels.Absolute path to the xml file with the kernels desc.
62  -d "<device>" Specify the target device for Face Detection (CPU, GPU, FPGA, or MYRIAD). Sample will look for a suitable plugin for device specified.
63  -d_ag "<device>" Specify the target device for Age Gender Detection (CPU, GPU, FPGA, or MYRIAD). Sample will look for a suitable plugin for device specified.
64  -d_hp "<device>" Specify the target device for Head Pose Detection (CPU, GPU, FPGA, or MYRIAD). Sample will look for a suitable plugin for device specified.
65  -d_em "<device>" Specify the target device for Emotions Detection (CPU, GPU, FPGA, or MYRIAD). Sample will look for a suitable plugin for device specified.
66  -n_ag "<num>" Specify number of maximum simultaneously processed faces for Age Gender Detection (default is 16).
67  -n_hp "<num>" Specify number of maximum simultaneously processed faces for Head Pose Detection (default is 16).
68  -n_em "<num>" Specify number of maximum simultaneously processed faces for Emotions Detection (default is 16).
69  -no_wait No wait for key press in the end.
70  -no_show No show processed video.
71  -pc Enables per-layer performance report.
72  -r Inference results as raw values.
73  -t Probability threshold for detections.
74 
75 ```
76 
77 Running the application with the empty list of options yields the usage message given above and an error message.
78 You can use the following command to do inference on a GPU with an example
79 pre-trained GoogleNet based SSD* available at https://software.intel.com/file/609199/download:
80 ```sh
81 ./interactive_face_detection -i <path_to_video>/inputVideo.mp4 -m <path_to_model>/ssd.xml -d GPU
82 ```
83 Notice that the network should be converted from the Caffe* (*.prototxt + *.model) to the Inference Engine format
84 (*.xml + *bin) first, by use of the ModelOptimizer tool
85 (https://software.intel.com/en-us/articles/OpenVINO-ModelOptimizer).

87 ### Sample Output
88 
89 The sample uses OpenCV to display the resulting frame with detections (rendered as bounding boxes and labels, if provided).
90 In the default mode the sample reports
91 * **OpenCV time**: frame decoding + time to render the bounding boxes, labels, and displaying the results.
92 * **Face Detection time**: inference time for the face Detection network.
93 
94 If Age Gender Head Pose or Emotion detections are enabled the additional info below is reported also:
95 * **Age Gender + Head Pose + Emotions Detection time**: combined inference time of simultaneously executed
96 age gender, head pose and emotion recognition networks.
